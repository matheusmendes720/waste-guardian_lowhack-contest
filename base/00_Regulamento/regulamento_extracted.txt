SECTION 2: TECHNICAL REQUIREMENTS (MANDATORY)
================================================================================

PLATFORM REQUIREMENTS:
1. Development Platform: Mendix Low-Code (100% mandatory)
2. Application Type: 100% Web Application
3. GenAI Integration: OpenAI (ChatGPT API provided by organizers)
4. Hosting: Mendix Cloud (Free Tier required)
5. Mendix Version: Studio Pro 10.24.0 or above

APPLICATION STRUCTURE REQUIREMENTS:
1. Minimum Pages: 3+ navigable pages
   - Required examples: Home page, Main functionality page, Results page
   
2. Data Persistence: Entities in Domain Model (CRUD operations)
   - Must implement Create, Read, Update, Delete functionality
   - Domain model must be properly defined
   
3. Microflows: At least 1 functional microflow or nanoflow
   - Must contain core business logic
   - Must be operational (not placeholder)
   
4. Responsiveness: Minimally responsive interface
   - Must work on different screen sizes
   - Mobile/tablet compatibility required
   
5. UI/UX Quality: Attractive interface
   - Professional appearance
   - User-friendly design

GENAI INTEGRATION REQUIREMENTS:
1. Must utilize provided OpenAI/ChatGPT API
2. Integration must be functional (not decorative)
3. Must demonstrate practical AI application
4. Proper error handling required
5. Clear user feedback mechanism

REQUIRED MENDIX MODULES:
- GenAI Commons module
- OpenAI Connector
- Encryption module
- Community Commons module

SECTION 6: JUDGING CRITERIA
================================================================================

SCORING SCALE:
- Range: 1 to 4 points per criterion
- 1 = Poor
- 2 = Fair
- 3 = Good
- 4 = Excellent

EVALUATION CATEGORIES:

1. POTENTIAL IMPACT (8.2.5.a)
   Weight: Standard
   Description: Evaluates the solution's potential impact on the proposed challenge
   Considerations:
   - Scale of impact
   - Feasibility
   - Sustainability
   - Real-world applicability

2. BUSINESS MODEL (8.2.5.b)
   Weight: Standard
   Description: Assesses business model, scalability, and application to solve challenge
   Considerations:
   - Revenue model viability
   - Market potential
   - Growth scalability
   - Implementation feasibility

3. CHALLENGE ADHERENCE (8.2.5.c)
   Weight: Standard
   Description: Evaluates whether solution effectively solves presented challenge
   Considerations:
   - Problem-solution fit
   - Alignment with ODS 9 (Industry, Innovation & Infrastructure)
   - Alignment with ODS 12 (Responsible Consumption & Production)
   - Theme relevance

4. SOLUTION INNOVATION (8.2.5.d)
   Weight: Standard
   Description: Assesses innovation in technology use and/or process change
   Considerations:
   - Novelty of approach
   - Creative use of technology
   - Differentiation from existing solutions
   - Process innovation

5. SOLUTION PRESENTATION (8.2.5.e)
   Weight: Standard
   Description: Evaluates clarity, completeness, and adherence to challenge requirements
   Considerations:
   - Communication quality
   - Demo effectiveness
   - Storytelling
   - Video production quality

6. TECHNICAL SOLUTION CRITERIA (8.2.5.f)
   Weight: Standard
   Description: Assesses application technical quality
   Components Evaluated:
   - Application functionality
   - Effective GenAI (ChatGPT API) usage
   - Connector/API integration quality
   - User experience (UX/UI) quality
   - Creativity in technology combination and implementation

TECHNICAL DELIVERABLES SCORING (8.2.2.b, 8.2.7):
Additional criteria for documentation evaluation:
- Interface minimally responsive and attractive
- Use of at least 1 functional microflow or nanoflow
- At least 3 navigable pages (home, main feature, results)
- Data persistence implementation (entities in domain model)

TIE-BREAKING CRITERIA (8.2.7):

Primary Tie-Breaker: Technical Deliverables Score
- Higher score in technical deliverables criteria wins
- Components:
  1. Responsive and attractive interface
  2. At least 1 functional microflow/nanoflow
  3. At least 3 navigable pages
  4. Data persistence implementation

Secondary Tie-Breaker: Innovation Score
- If still tied, higher "Solution Innovation" score wins

Final Tie-Breaker: Organizing Committee Decision
- If still tied after both criteria, committee decides

SECTION 8: THEME & CHALLENGE DESCRIPTION
================================================================================

OFFICIAL CHALLENGE STATEMENT:
"How can we develop innovative solutions that encourage conscious consumption 
and responsible production practices, reducing waste, promoting efficient 
resource use, and stimulating more sustainable models throughout the value chain?"

MACRO THEME:
The current consumption and production model generates significant environmental 
impacts. Rethinking how we produce, consume, and discard products is one of the 
great challenges of our time.

ALIGNMENT:
United Nations Sustainable Development Goals:
- ODS 9: Industry, Innovation and Infrastructure
- ODS 12: Responsible Consumption and Production

PERMITTED SOLUTION AREAS:
Teams may develop prototypes and ideas that use technology to:

1. Accelerate digital transformation in industry
2. Make production processes more efficient and intelligent
3. Reduce waste and optimize resource use
4. Encourage more responsible production and consumption practices
5. Create digital solutions that support more sustainable production chains
